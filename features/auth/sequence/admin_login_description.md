## 필수 정보

### 1. 참여자 (Participants/Actors)
| 참여자 | 설명 | 기술 스택 |
|--------|------|-----------|
| 관리자 (Admin) | 시스템 관리자 | 브라우저 |
| 관리자 프론트엔드 (Admin Frontend) | 관리자 웹 클라이언트 | Next.js + Nginx |
| 백엔드 (Backend) | API 서버 | Spring + Spring Security |
| DB (Database) | 데이터 저장소 | - |

### 2. 시나리오/흐름 설명
- 관리자 로그인 프로세스
- 관리자 토큰 갱신 프로세스
- 관리자 로그아웃 프로세스

### 3. 메시지 흐름 순서

#### 3-1. 관리자 로그인 흐름
```
1. 관리자 → 관리자 프론트엔드: 관리자 로그인 페이지 접속 (/admin/login)
2. 관리자 → 관리자 프론트엔드: 로그인 폼 제출 (이메일, 비밀번호)
3. 관리자 프론트엔드 → 백엔드: 관리자 로그인 API 요청 (POST /api/admin/auth/login)
4. 백엔드 → DB: 관리자 계정 조회 (이메일로 검색)
5. DB → 백엔드: 관리자 정보 반환
6. 백엔드: 비밀번호 검증 (BCrypt) + 관리자 권한(ROLE_ADMIN) 확인
7. [조건] 인증 성공 시:
   - 백엔드: Access Token 생성 (JWT, ROLE_ADMIN 포함)
   - 백엔드: Refresh Token 생성 (JWT)
   - 백엔드 → DB: Refresh Token 저장
   - 백엔드 → 관리자 프론트엔드: 로그인 성공 응답 (토큰을 Cookie에 설정)
   - 관리자 프론트엔드 → 관리자: 관리자 대시보드로 이동
8. [조건] 인증 실패 시:
   - 백엔드 → 관리자 프론트엔드: 에러 응답 (401 Unauthorized)
   - 관리자 프론트엔드 → 관리자: 로그인 실패, 에러 메시지 표시
```

#### 3-2. 관리자 토큰 갱신 흐름
```
1. 관리자 프론트엔드: Access Token 만료 감지
2. 관리자 프론트엔드 → 백엔드: 토큰 갱신 요청 (POST /api/admin/auth/refresh)
3. 백엔드 → DB: Refresh Token 유효성 확인
4. DB → 백엔드: Refresh Token 정보 반환
5. [조건] 유효한 경우:
   - 백엔드: 새로운 Access Token 생성
   - 백엔드 → 관리자 프론트엔드: 200 OK + Set-Cookie (새 Access Token)
6. [조건] 무효한 경우:
   - 백엔드 → 관리자 프론트엔드: 401 Unauthorized
   - 관리자 프론트엔드 → 관리자: 관리자 로그인 페이지로 리다이렉트
```

#### 3-3. 관리자 로그아웃 흐름
```
1. 관리자 → 관리자 프론트엔드: 로그아웃 버튼 클릭
2. 관리자 프론트엔드 → 백엔드: 로그아웃 요청 (POST /api/admin/auth/logout)
3. 백엔드 → DB: Refresh Token 삭제
4. DB → 백엔드: 삭제 완료
5. 백엔드 → 관리자 프론트엔드: 200 OK + Set-Cookie (토큰 삭제)
6. 관리자 프론트엔드 → 관리자: 관리자 로그인 페이지로 이동
```

## 추가 기술 정보

### 인증/인가 방식
| 항목 | 설명 |
|------|------|
| 관리자 로그인 페이지 | /admin/login (별도 페이지) |
| 권한 확인 | JWT 토큰에 관리자 권한(ROLE_ADMIN) 포함 |
| API 접근 제어 | /api/admin/** 경로는 관리자 권한 필수 |
| 비밀번호 암호화 | BCrypt |

### API 엔드포인트
| 기능 | Method | Endpoint | 설명 |
|------|--------|----------|------|
| 관리자 로그인 | POST | /api/admin/auth/login | 관리자 전용 로그인 |
| 토큰 갱신 | POST | /api/admin/auth/refresh | Access Token 갱신 |
| 로그아웃 | POST | /api/admin/auth/logout | 로그아웃 처리 |

### 에러 케이스
| HTTP Status | 설명 | 발생 조건 |
|-------------|------|-----------|
| 401 Unauthorized | 인증 실패 | 잘못된 이메일/비밀번호, 만료된 토큰 |
| 403 Forbidden | 권한 없음 | 관리자가 아닌 사용자가 접근 |
