## 필수 정보

### 1. 참여자 (Participants/Actors)
| 참여자 | 설명 | 기술 스택 |
|--------|------|-----------|
| 사용자 (User) | 서비스 이용자 | 브라우저 |
| 프론트엔드 (Frontend) | 웹 클라이언트 | Next.js + Nginx |
| 백엔드 (Backend) | API 서버 | Spring + Spring Security |
| DB (Database) | 데이터 저장소 | - |

### 2. 시나리오/흐름 설명
- LECTURE 목록 조회 프로세스
- LECTURE 상세 조회 프로세스

### 3. 메시지 흐름 순서

#### 3-1. LECTURE 목록 조회 흐름
```
1. 사용자 → 프론트엔드: 강의 목록 페이지 요청
2. 프론트엔드 → 백엔드: 목록 조회 API 요청 (GET /api/lectures?page=0&size=4&category={category}&sort=rating)
3. 백엔드 → DB: LECTURE 목록 조회 (페이지네이션, 필터, 정렬 적용)
4. DB → 백엔드: LECTURE 목록 데이터 반환
5. 백엔드 → 프론트엔드: 200 OK (LECTURE 목록 + 페이지 정보)
6. 프론트엔드 → 사용자: 강의 목록 화면 표시
```

#### 3-2. LECTURE 상세 조회 흐름
```
1. 사용자 → 프론트엔드: 강의 상세 페이지 요청 (강의 클릭)
2. 프론트엔드 → 백엔드: 상세 조회 API 요청 (GET /api/lectures/{id})
3. 백엔드 → DB: LECTURE 단건 조회 (ID로 검색)
4. DB → 백엔드: LECTURE 데이터 반환
5. [조건] 조회 성공 시:
   - 백엔드 → 프론트엔드: 200 OK (LECTURE 상세 데이터)
   - 프론트엔드 → 사용자: 강의 상세 화면 표시
6. [조건] 조회 실패 시 (LECTURE 없음):
   - 백엔드 → 프론트엔드: 404 Not Found
   - 프론트엔드 → 사용자: 에러 페이지 또는 메시지 표시
```

## 추가 기술 정보

### 인증/인가 방식
| 항목 | 설명 |
|------|------|
| LECTURE 목록 조회 | 인증 불필요 (비로그인 허용) |
| LECTURE 상세 조회 | 인증 불필요 (비로그인 허용) |

### 조회 옵션
| 항목 | 설명 |
|------|------|
| 페이지네이션 | page, size=4 |
| 필터 | 카테고리 (category) |
| 정렬 | 별점순 (rating) |

### API 엔드포인트
| 기능 | Method | Endpoint | 설명 |
|------|--------|----------|------|
| 목록 조회 | GET | /api/lectures | 페이지네이션, 필터, 정렬 지원 |
| 상세 조회 | GET | /api/lectures/{id} | ID로 단건 조회 |

### 응답 데이터
| 항목 | 설명 |
|------|------|
| 목록 조회 | LECTURE 목록 + 페이지 메타 정보 (totalPages, totalElements 등) |
| 상세 조회 | LECTURE 상세 데이터 |

### 에러 케이스
| HTTP Status | 설명 | 발생 조건 |
|-------------|------|-----------|
| 404 Not Found | LECTURE 없음 | 존재하지 않는 ID로 상세 조회 시 |
