## ✅ 정리

- `yaml` 파일 꼼꼼히 써놨기 때문에 CI/CD 흐름에 대해서 잘 모르겠으면 yaml 파일 살펴보고 민지에게 질문하기
- 강의 관련 로직은 끝인가? → 강의 수정 필요! (삭제는 없음) 강의 수정도 다 수정하는 게 아니라 이름 등 몇 개만 수정 → 이거는 데이터 맞춰보자
- 어쨌든 우리가 모든 기능이 정상적으로 돌아가려면 관리자가 필요하긴 하다 -> 관리자는 어쨌든 승인하는 게 제일 크니까 일단은 그냥 승인됐다고 치고 만들자. 다음 주에 관리자 페이지 만든 다음에 빨리 붙이자.
- 1차 기간이 이번 주말까지니까 관리자 페이지 말고는 다 완성을 해놓고, 관리자 페이지를 phase 2에서 완성하자. phase 2에서는 AWS Kinesis를 달고 시작하자. 그래야 개발 편할 것
- 우선적으로 AWS SES, S3을 달자. 그러면 DB 관련된 것은 끝!
- git submodule의 경우 complict가 났는데 그걸 어떻게 수정하는가? 수정하는 곳이 없었다 → `resources/config` 들어가면(git submodule 들어가면) 생길 것! config에 대해서도 slack 알림이 필요할 것 같음
